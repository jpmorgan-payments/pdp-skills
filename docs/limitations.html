Limitations & Disclaimers

Important information about what these skills can and cannot do.

Limitations

Guidance Only — Not a Live API Client

These skills provide procedural guidance to your agent and generate code in your project. They do not make live API calls, connect to any JPM environment, or process real transactions. All generated code is illustrative and must be tested against the actual API in CAT before being used in PROD.

Point-in-Time Documentation

While the skill references draw from the J.P. Morgan Payments Developer Portal, the skill itself may contain outdated docs or references. Always cross-reference with the latest official JPM documentation before deploying to production.

No Sandbox or Test Credentials

These skills do not include or provide sandbox credentials, API keys, or test merchant accounts. You must obtain your own credentials directly from J.P. Morgan — reach out to your Relationship Manager (RM).

Limited API Surface

The skills currently cover:

  • Getting started with JPMorgan Payments APIs (jpm-integrations-get-started)
  • OAuth 2.0 + JWT authentication (jpm-oauth)
  • Checkout — Drop-in UI and Hosted Payments Page (jpm-merchant-integrations)
  • Online Payments — card payments lifecycle (jpm-merchant-integrations)

More APIs are coming soon, including Tokenization, Reporting, and others.

AI-Generated Code Requires Review

Do not deploy AI-generated code without thorough review and testing. Even with these skills loaded, the agent can produce code that is incorrect, incomplete, or insecure. Possible issues include:

  • Incorrect field values, formats, or enum constants
  • Missing required headers, parameters, or authentication steps
  • Outdated or invalid endpoint URLs
  • Security vulnerabilities (e.g., exposing credentials client-side, improper token storage)
  • Logic errors in payment flows (e.g., capturing before authorization)

You are responsible for validating all generated code. Always test against JPM's CAT environment, cross-reference with official documentation, and have your code reviewed by a team member before moving to production. Treat AI output as a starting point — not as production-ready code.

No Guaranteed Trigger

The skills rely on the AI agent's description-matching heuristics to determine when each one is relevant. In some cases, a skill may not load automatically if your prompt is ambiguous. Use the explicit slash command to force loading when needed.

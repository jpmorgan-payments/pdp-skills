Usage & Prompts

How the skills drive the conversation — and how to invoke the one you need.

How the Skills Work

The skills do the heavy lifting. You start with one short prompt — usually a slash command or a single natural-language sentence — and the skill takes it from there.

From that point on, most of your input is picking options from a multiple-choice menu the skill presents. You almost never have to write paragraphs. Each skill asks structured questions, generates code based on your answers, and hands off to the next skill when you're ready.

Minimal prompting by design. The skills are written to ask the right questions in the right order. Your job is to pick the answers — the skill knows what to do with them.

Conversational Flow

Here's what each skill walks you through. The triggers below are slash commands; you can also use natural language (see Invoking a Specific Skill below).

jpm-integrations-get-started

The entry point. Use this when you're new to J.P. Morgan Payments or want the full guided experience.

You type once /jpm-integrations-get-started

The skill then asks:

  1. "Do you acknowledge the disclaimer about AI-generated code?" Options: Yes — proceed · No — exit
  2. "What are you trying to do with J.P. Morgan Payments?" Options: Migrate from another provider · Start a new JPM integration
  3. "Have you received your clientId, certificate, and private key from your J.P. Morgan Relationship Manager?" Options: Yes — I have all three · No — not yet
  4. (Automatic) The skill detects your .env state and either appends missing keys or generates a sample .env.example — no question asked unless input is needed.
  5. "Would you like me to implement the JPM OAuth flow now?" Options: Yes — generate auth code · No — I'll do it later
Outcome: A project ready to wire up authentication, with all the environment variables in place. If you say yes to the last question, it hands off to jpm-oauth automatically.
Typical user input after the trigger: 4 button clicks.

jpm-oauth

Generates a working OAuth module in your project. Use directly when you already have credentials.

You type once /jpm-oauth Generate auth in my Python project

The skill then asks:

  1. (Automatic) Verifies your credentials are in context (clientId, cert, key, thumbprint). If anything is missing, it tells you what to fill in.
  2. "Which JPM environment are you targeting?" Options: CAT (test) · PROD · Both — generate code that switches via env var
  3. "How long should JWTs be valid?" Options (PROD): 8 hours (recommended) · 1 hour · Custom  ·  Options (CAT): 6 months · 8 hours · Custom
  4. (Automatic) Detects your language and HTTP/JWT libraries from your project manifest and announces the choice. Asks only if it can't detect.
  5. "Where should I put the auth module?" Options: Inside the project source tree · A new top-level folder · Custom path
  6. "Auth is in place. Want to integrate a J.P. Morgan Payments API now?" Options: Yes — pick an API to integrate (Checkout or Online Payments) · No — I'll come back later
Outcome: A language-idiomatic auth module that exports a cached getAccessToken() function (or its language equivalent), ready to use with any J.P. Morgan Payments API call. If you say yes to the last question, it hands off to jpm-merchant-integrations.
Typical user input after the trigger: 3 button clicks (most prompts auto-resolve).

jpm-merchant-integrations

Integrates an API in your project. Use directly when you already have auth wired up — for your first API or to expand by adding another.

You type once /jpm-merchant-integrations Integrate Checkout in my Node project

The skill then asks:

  1. (Automatic) Confirms your auth module is ready to import. If not, points you back to jpm-oauth.
  2. "Which API would you like to integrate first?" Options: Checkout — drop-in payment UI · Online Payments — direct API processing
  3. "Briefly describe what you're building or migrating." One free-text sentence — e.g. "a single-product checkout page" or "migrating from Stripe Elements". Used to tailor the generated code.
  4. (Automatic) Loads the relevant API reference, generates integration code that imports your auth module, and provides a CAT smoke-test recipe.
  5. "That integration looks good. Want to add another JPM API now?" Options: Yes — pick another · No — I'm done for now
Outcome: A working integration in your project for the API you chose, with a clear path to expand by adding more APIs over time.
Typical user input after the trigger: 1 button click + 1 short sentence.

Invoking a Specific Skill

Each skill works on its own. You don't have to start at jpm-integrations-get-started — invoke the one you need directly. There are two ways.

Slash Command (Recommended)

Explicit, deterministic, works the same in every Agent harness. Format: /skill-name followed by a brief description of what you want.

Get Started
"/jpm-integrations-get-started I'm migrating from Stripe"
OAuth — Specific Stack
"/jpm-oauth Generate auth in Python with httpx and pyjwt"
OAuth — Both Environments
"/jpm-oauth Generate code that switches between CAT and PROD via env var"
Integrations — First API
"/jpm-merchant-integrations Integrate Checkout with the drop-in UI"
Integrations — Expand
"/jpm-merchant-integrations Add Online Payments to my existing Checkout integration"
Integrations — Specific Feature
"/jpm-merchant-integrations Add 3-D Secure to my Online Payments card flow"

Natural Language

Once you've used the skills a few times, you can skip the slash command and just describe what you want. The agent matches your prompt to each skill's description field and loads the right one. Less reliable than a slash command — fall back to slash if the wrong skill (or no skill) loads.

Triggers jpm-integrations-get-started
"I'm new to J.P. Morgan Payments — walk me through what I need."
Triggers jpm-oauth
"Generate a JPM OAuth module in my project with token caching."
Triggers jpm-merchant-integrations
"I have JPM auth set up. Now help me integrate Checkout."
Triggers jpm-merchant-integrations (expansion)
"Add JPM Online Payments to my existing Checkout integration."

Tips for Better Results

  • Default to the slash command. It's the most reliable way to load the right skill — natural-language matching can miss when your prompt is ambiguous.
  • State your starting point in the trigger. Saying "I already have auth" or "I'm starting from scratch" helps the skill skip phases you don't need.
  • Mention your stack. Say "in Node.js" or "in Python with httpx" to get language-appropriate code generation.
  • Pick options as offered. The skills present multiple-choice menus for almost every decision — you don't need to compose long answers.
  • Be specific about the API. Mention "Checkout" or "Online Payments" to help the agent route to the correct reference file.
  • Smoke test in CAT first. Each integration includes a CAT test recipe. Run it before flipping to PROD.
  • If you get stuck, ask the agent "what skill am I in?" or "what step are we on?" — the skills are written to handle re-entry and recovery questions cleanly.

Setup & Installation

Install the skills for your agent in under a minute.

Claude Code

The fastest path is the Claude Code plugin manager — this repository ships as a plugin with a marketplace manifest at the root.

Plugin Install (Recommended)

Inside Claude Code, run:

/plugin marketplace add <github-repo-url>
/plugin install jpm-payments-skills@jpm-payments-skills

Replace <github-repo-url> with this repository's URL. After install, run /plugin to confirm it's listed and /skills to see the three jpm-* skills.

To update later:

/plugin update jpm-payments-skills

Manual Install (Alternative)

If you'd rather copy the skills directly into your project, Claude Code auto-discovers any skills under .claude/skills/: 

your-project/
├── .claude/
│   └── skills/
│       ├── jpm-integrations-get-started/
│       ├── jpm-oauth/
│       └── jpm-merchant-integrations/
└── ...

Clone this repository and copy the skill folders:

git clone <github-repo-url> /tmp/jpm-skills
mkdir -p .claude/skills
cp -r /tmp/jpm-skills/skills/jpm-* .claude/skills/

For a user-level install (every Claude Code project sees the skills), copy into ~/.claude/skills/ instead.

Tip: You can invoke any skill explicitly via its slash command (for example, /jpm-integrations-get-started) to force it to load regardless of prompt content. Otherwise, any prompt that matches the skill's description field triggers it automatically.

GitHub Copilot

GitHub Copilot loads agent skills from .github/skills/ at the root of your project. No extra configuration is needed.

Steps

  1. Copy the skill folders into your project: 
    your-project/
├── .github/
│   └── skills/
│       ├── jpm-integrations-get-started/
│       ├── jpm-oauth/
│       └── jpm-merchant-integrations/
└── ...
  2. Open your project in VS Code or a JetBrains IDE with GitHub Copilot Chat enabled.
  3. Start asking questions. Copilot loads the relevant skill automatically when your prompt matches the skill's description.

Clone-and-copy commands:

git clone <github-repo-url> /tmp/jpm-skills
mkdir -p .github/skills
cp -r /tmp/jpm-skills/skills/jpm-* .github/skills/
Tip: You can invoke a skill explicitly via its slash command (for example, /jpm-merchant-integrations) which forces it to load. Alternatively, any prompt that matches the skill's description field triggers it automatically.

Cursor

Cursor reads the same Agent Skills open format. Place the skill folders in .cursor/skills/ at the root of your project:

your-project/
├── .cursor/
│   └── skills/
│       ├── jpm-integrations-get-started/
│       ├── jpm-oauth/
│       └── jpm-merchant-integrations/
└── ...

Clone-and-copy commands:

git clone <github-repo-url> /tmp/jpm-skills
mkdir -p .cursor/skills
cp -r /tmp/jpm-skills/skills/jpm-* .cursor/skills/

Restart Cursor or reload the workspace so the agent picks up the new skills, then trigger one with a natural-language prompt.

Note: Cursor's support for the Agent Skills open format is evolving. If your version doesn't recognize the directory above, refer to the current Cursor documentation for the canonical path. The SKILL.md contents are portable; only the directory the harness scans differs.

Other Agent Harnesses

The skills are written in the open Agent Skills format — a SKILL.md file with YAML frontmatter (name, description) plus markdown body. Any Agent harness that supports this format can load and use these skills.

Generic Install Steps

  1. Clone this repository.
  2. Copy the skills/jpm-*/ directories into the location your harness scans for agent skills (consult your harness's documentation for the canonical path).
  3. Restart the harness or reload the workspace so it picks up the new skills.
  4. Trigger a skill with its slash command (/jpm-integrations-get-started, /jpm-oauth, or /jpm-merchant-integrations), or with a natural-language prompt that matches its description.

Each skill folder is self-contained — the SKILL.md plus optional references/, templates/, examples/, and scripts/ subfolders that the skill points to during execution. As long as your harness can read SKILL.md and follow links to local files, everything in the directory is portable.

Note: If your harness doesn't auto-discover skills from a directory, you can pass the SKILL.md content directly as system context or instructions. The conversational flow still works — only the auto-trigger behavior may differ from native harnesses.
